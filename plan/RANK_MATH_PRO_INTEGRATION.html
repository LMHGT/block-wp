LMHG Rank Math Pro Integration Contract

Rank Math Pro becomes the editing and output authority for standard SEO after an administrator-approved, conflict-preserving one-time handoff. LMHG Site Core remains the source of page relationships, publication readiness, CTA variants, organization rules, provenance, and fallback behavior.

Content model clarification

The live WordPress page is stored as Gutenberg post_content, which Rank Math analyzes natively. wp2026-page-data.json is a versioned import/source artifact, not a second live page store and not something Rank Math should parse on every request. LMHG's editor bridge uses Rank Math's rank_math_content JavaScript filter only for content assembled outside post_content: taxonomy-driven FAQs and related pages, team/review sections, and CTA term copy.

Supported integration hooks

  • rank_math_content and rank_math_title: editor analysis of plugin/custom-field content.
  • rank_math/researches/tests and rank_math/content_ai/research_tests: adjust only tests that are invalid for an LMHG page family.
  • rank_math/metabox/post/meta_keys and rank_math/metabox/post/values: expose LMHG audit/provenance data in Gutenberg if needed.
  • rank_math/vars/register_extra_replacements: optional stable variables backed by LMHG taxonomy data.
  • rank_math/frontend/title, description, robots, and canonical: last-resort organization rules, not ongoing two-way sync.
  • rank_math/opengraph/{network}/image: approved global fallback share image.
  • rank_math/json_ld and related schema filters: merge unique LMHG entities and prevent duplicates.
  • rank_math/sitemap/entry: canonical/private/exclusion enforcement.

Pro capabilities to evaluate deliberately

Unlimited/reorderable focus keywords; Quick Edit and CSV metadata workflows; multiple schema types; Custom Schema Builder, reusable templates, variables, and display conditions; Analytics and Rank Tracker; 404 monitoring and redirect categories/schedules/CSV; Local SEO/KML if the physical-location model warrants it; Image SEO; Role Manager; SEO Analyzer; AI Link Genius; AI Visibility; Content AI; llms.txt; and Rank Math MCP tools. Content AI is a separate editorial assistance surface and must not silently replace approved clinical or legal copy.

Leave unrelated integrations disabled unless the content model changes: WooCommerce/EDD, bbPress/BuddyPress, AMP, Podcast, News Sitemap, and Video Sitemap. Rank Math is modular; “use Pro fully” means using every relevant module with one clear owner, not enabling every module indiscriminately.

Rank Math's MCP surface can retrieve SEO metadata, schema, post links, and site-wide link health, and newer Pro releases expose competitor audits and richer link reports. Mutation tools such as rank-math/fix-site-seo must remain disabled or human-approved because they can alter indexing, permalink, sitemap, schema, robots, tagline, and focus-keyword settings.

Activation gates

  1. Obtain and hash the licensed Free/Pro packages; record exact versions and verify their compatibility together. Rank Math Pro is an add-on and both Free and Pro must remain installed and active, with Free updated first. As of 2026-07-11, WordPress.org reports Free 1.0.273 and Rank Math reports Pro 3.0.116; the previously named 1.0.271.1/3.0.114 pair is no longer current.
  2. Back up database, source, runtime, uploads, settings, redirects, and current public metadata.
  3. Install inactive, inspect modules/settings, then activate on development only.
  4. Run LMHG handoff dry-run. Resolve every pre-existing Rank Math conflict; never overwrite silently.
  5. Verify Gutenberg analysis sees ordinary blocks once and server-rendered LMHG sections once.
  6. Validate all canonical routes for one title, description, canonical, robots policy, social set, and one coherent JSON-LD graph.
  7. Verify sitemap membership, route blocking, redirects, 404 logging, permissions, Quick Edit/CSV keyword order, and deactivation fallback.
  8. Connect license, Rank Math account, Search Console, Analytics, and Rank Tracker only during the approved go-live phase.

Current gate: Rank Math is not installed. The LMHG adapter is dormant. Installation, licensing, account connections, data writes, and redirect ownership transfer require an explicit go-live decision.
