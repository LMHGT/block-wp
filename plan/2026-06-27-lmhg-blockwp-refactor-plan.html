Execution Phases

Phase 0: Repo And Source Provenance

  • Keep the copied handoff under docs/superpowers/plans/.
  • Create docs/source-provenance.md with Astro path, branch, SHA, LMHG plugin version, health checks, and extraction time.
  • Add a no-edit note for the Astro worktree.

Phase 1: Agentic WordPress Scaffold

  • Implement the supplied handoff with LMHG-owned names.
  • Install project-local WordPress Agent Skills.
  • Create the block theme, site-core plugin, Blueprint, and verification scripts.
  • Stop with placeholder-safe LMHG content. This is not a migration yet.

Phase 2: Astro Staging Inventory Export

  • Read route, copy, relationship, SEO, team, article, asset, and redirect sources from Astro without editing them.
  • Generate source route, design, and asset manifests under data/lmhg/.
  • Give every route and redirect an explicit migration status.

Phase 3: WordPress Content Model And Import Contract

  • Define WordPress representation for each page family.
  • Keep SEO/schema/graph state in plugin-owned structured files or options.
  • Build an idempotent local seed/import command.

Phase 4: LMHG Block Theme Templates

  • Translate LMHG visual tokens into theme.json.
  • Build header, footer, and page-family templates with crawlable content.
  • Avoid tabs, carousels, generic imagery, page-builder patterns, and decorative filler.

Phase 5: Site-Core Plugin Parity

  • Generate metadata, JSON-LD, breadcrumbs, related pages, redirects, and canonical behavior from structured manifests.
  • Add validation for missing meta, missing H1, bad links, unsupported city service-area exposure, and placeholder copy.

Phase 6: Workbench And Editing Parity

  • Preserve the rule that Standard mode comes from rendered preview markers, not raw database inventory.
  • Add or document data-lmhg-edit-field marker output for visible editable content.
  • Defer live Workbench write integration until marker parity is proven.

Phase 7: Parity Verification

npm run check:static
npm run verify:lmhg-static
npm run wp-env:start
npm run wp-env:seed
npm run verify:site
npm run verify:lmhg-routes
npm run verify:lmhg-redirects
npm run verify:lmhg-markers
npm audit --omit=dev

Route parity must compare status, redirects, canonical URL, title, meta description, H1, CTA, breadcrumbs, related links, FAQ presence, JSON-LD, unsupported service-area exposure, and placeholder copy.

Phase 8: Staging And Cutover Decision

  • Only start after Phases 1 through 7 pass.
  • Confirm hosting, domain, backup, editor access, and rollback models.
  • Require explicit approval before public DNS, Cloudflare, or production changes.
